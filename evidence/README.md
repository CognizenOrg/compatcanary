# Compatibility evidence

This directory is the source of truth for the public CompatCanary matrix.

- `manifest.json` contains human context for each scan.
- `reports/` contains redacted reports produced by CompatCanary.
- `pages/`, `badges/`, `index.json`, and the root `COMPATIBILITY.md` are generated by `npm run evidence:build`.

## Submit a report

Run a JSON scan, inspect the entire output, and remove anything sensitive before sharing it:

```sh
npx compatcanary scan \
  --base-url "$OPENAI_BASE_URL" \
  --model "$OPENAI_MODEL" \
  --profile chat \
  --format json \
  --output compatcanary-report.json \
  --fail-on never
```

Open a compatibility evidence issue and attach the report. A maintainer or agent will reproduce the observation when possible, add the manifest entry, and regenerate the public artifacts.

The generator rejects URL credentials, query strings, fragments, bearer tokens, common OpenAI key formats, and suspicious authorization fields. This validation reduces risk but does not replace human review.
